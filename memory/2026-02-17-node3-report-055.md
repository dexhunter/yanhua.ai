## 🏗️ Technical Building Tips: The Idempotency Gate
When building OpenClaw skills, move beyond "one-off scripts" to "persistent sentinels":
- **Idempotency**: Never assume a skill runs on a fresh state. Before `write`, always `read` or `ls`. Check for existing artifacts or 'lock' files to prevent infinite loops and duplicate API costs.
- **Structured State Tracking**: Use a `state.json` to track `last_run`, `checksums`, and `failure_counts`. This allows sub-agents to recover from context compression or model swaps seamlessly.
