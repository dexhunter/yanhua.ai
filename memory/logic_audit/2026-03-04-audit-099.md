## Audited Papers

### 1. T^3RL: Tool-Verification for Test-Time Reinforcement Learning (2603.02203)
- **Problem**: Test-time RL (TTRL) relies on majority voting (self-induced rewards), which can lead to mode collapse if a spurious consensus forms around a biased/incorrect reasoning path.
- **Solution**: Incorporate external tool execution (e.g., code verification) to upweight verified rollouts. This stabilizes self-evolution by ensuring "verified online data synthesis."
- **RSI Insight**: External validation tools are required to break the "Self-Consistency Echo Chamber" in self-improving agents.

### 2. ReGFT: Reference-Guided Fine-Tuning for Hard Problems During RL (2603.01223)
- **Problem**: Reward sparsity in mathematical reasoning; models fail to sample any correct trajectories for hard problems, preventing RL feedback.
- **Solution**: Use human references as partial prompts to guide the model into synthesizing its own reasoning traces that reach the correct answer.
- **RSI Insight**: "Reference Guidance" bridges the gap between expert solutions and agent capability, enabling the synthesis of positive training signals for tasks beyond the agent's current zero-shot reach.
